工作区、暂存区和版本库的区别
    • 工作区：就是在电脑里能看到的目录。
    • 暂存区：英文叫stage, 或index。一般存放在 ".git目录下" 下的index文件（.git/index）中，所以我们把暂存区有时也叫作索引（index）
    • 版本库：工作区有一个隐藏目录.git，这个不算工作区，而是Git的版本库
    • 版本库里包括暂存区，分支， Git的对象库
      git的常用命令
    • git add -A 暂存区与工作区一致
    • git add -u 暂存区删除文件及更改文件
    • git add . 把更改的全部代码add到暂存区
    • git add 1.txt 2.txt add多个文件 用空格隔开
    • git add *.txt 通配符 add所有txt文件
    • git commit -m "" 提交更改,把暂存区的所有内容提交到当前分支
    • git commot --amend 更改之前一次commit的描述信息
    • git commot -a -m "" 把暂存区的所有被修改或者已删除的且已经被git管理的文档提交提交到分支，须输入描述信息（可省略 git add 过程）
    • git push 提交到远程仓库
    • git status 查看文件更改状态
    • git diff 查看工作区和暂存区的区别
    • git rm 工作区删除文件，暂存区也需要删除 多个文件删除用空格隔开
    • git reset HEAD 删除已git add的所有文件
    • git reset HEAD 删除已git add的所有文件
    • git stash 遇到本地有冲突文件 将工作区的修改暂时缓存起来 然后pull代码 再git stash pop 命令恢复之前缓存的工作目录
    • git stash list 查看提交到暂存区的文件列表
    • git stash clear 清空你所有的内容
    • git stash drop stash@{0} 这是删除第一个队列
    • git stash show 查看stash文件的增减记录
    • git stash show -p/--patch 可以查看特定stash的全部diff
